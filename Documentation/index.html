Introduction

A React Native Online Shopping App with complete backend integration.

App is built using Redux and is performance optimised which helps you to develop world-class application experiences on native platforms. This gives your app a consistent look and feel with the rest of the platform ecosystem, and keeps the quality bar high.

  • React Version: 18.2.0
  • React Native Version: 0.72.3
  • React Navigation Version: 6.1.7
  • Redux Version: 4.2.1

The app includes all essential screen along with many customisable options through a configuration file.

In this documentation we will explore everything in this app so you can get started with the app.

Let's get started.

Requirements

NodeJS

Make sure you have a recent version (16 or newer) of Node installed globally.

We recommend installing Node and Watchman using Homebrew. Run the following commands in a Terminal after installing Homebrew:

brew install node
brew install watchman

If you have already installed Node on your system, make sure it is Node 16 or newer.


For IOS View
Xcode & CocoaPods

The easiest way to install Xcode is via the Mac App Store. Installing Xcode will also install the iOS Simulator and all the necessary tools to build your iOS app.

If you have already installed Xcode on your system, make sure it is version 10 or newer.

Installing an iOS Simulator in Xcode

To install a simulator, open Xcode > Preferences… and select the Components tab. Select a simulator with the corresponding version of iOS you wish to use.

CocoaPods

CocoaPods is one of the dependency management system available for iOS. It is built with Ruby and you can install it using the version of Ruby you configured with in the previous steps.


For Android View
Java Development Kit

We recommend installing the OpenJDK distribution called Azul Zulu using Homebrew. Run the following commands in a Terminal after installing Homebrew:

brew tap homebrew/cask-versions
brew install --cask zulu11

The Zulu OpenJDK distribution offers JDKs for both Intel and M1 Macs. This will make sure your builds are faster on M1 Macs compared to using an Intel-based JDK.

If you have already installed JDK on your system, we recommend JDK 11. You may encounter problems using higher JDK versions.

Android development environment

please make sure to carefully follow the next few steps.

1. Install Android Studio

Download and install Android Studio. While on Android Studio installation wizard, make sure the boxes next to all of the following items are checked:

  • Android SDK
  • Android SDK Platform
  • Android Virtual Device

Then, click "Next" to install all of these components.

Once setup has finalized and you're presented with the Welcome screen, proceed to the next step.

2. Install the Android SDK

Android Studio installs the latest Android SDK by default. Building a React Native app with native code, however, requires the Android 13 (Tiramisu) SDK in particular. Additional Android SDKs can be installed through the SDK Manager in Android Studio.
To do that, open Android Studio, click on "More Actions" button and select "SDK Manager".

Select the "SDK Platforms" tab from within the SDK Manager, then check the box next to "Show Package Details" in the bottom right corner. Look for and expand the Android 13 (Tiramisu) entry, then make sure the following items are checked:

  • Android SDK Platform 33
  • Intel x86 Atom_64 System Image or Google APIs Intel x86 Atom System Image or (for Apple M1 Silicon) Google APIs ARM 64 v8a System Image

Next, select the "SDK Tools" tab and check the box next to "Show Package Details" here as well. Look for and expand the "Android SDK Build-Tools" entry, then make sure that 33.0.0 is selected.

Finally, click "Apply" to download and install the Android SDK and related build tools.

3. Configure the ANDROID_HOME environment variable

The React Native tools require some environment variables to be set up in order to build apps with native code.

Add the following lines to your ~/.zprofile or ~/.zshrc (if you are using bash, then ~/.bash_profile or ~/.bashrc) config file:

export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools

Run source ~/.zprofile (or source ~/.bash_profile for bash) to load the config into your current shell. Verify that ANDROID_HOME has been set by running echo $ANDROID_HOME and the appropriate directories have been added to your path by running echo $PATH.

React Native Command Line Interface

React Native has a built-in command line interface. Rather than install and manage a specific version of the CLI globally, we recommend you access the current version at runtime using npx, which ships with Node.js. With npx react-native , the current stable version of the CLI will be downloaded and executed at the time the command is run.

Configuration

Open config.js file in root level

  • Configure api url
    • Replace apiUrl with the hosting url on which your Node API is running
  • Configure google:
    • Replace apiKey with your google account api key
  • Configure razor payment:
    • Replace accessKeyId with your razor account access key
  • Configure Amazon S3:
    • Replace accessKeyId with your Amazon S3 access key id
    • Replace secretAccessKey with your Amazon S3 secret access key
    • Replace region with your Amazon S3 region
    • Replace bucket with your Amazon S3 bucket name

Customize

Customizing the App will be a cakewalk for you. That is due to the fact, we provide you well prepared code. This makes it quite simple to be able to dig through the code and hence without problems customizable.

Change splash image
For IOS

You should upload ios splash images to ios/GMart/Images.xcassets/splash.imageset/ with respective image name and size

For Android

You should upload android splash images to android/app/src/main/res/drawable_folder_name(drawable-hdpi, drawable-mdpi) with respective image name and size

Change app icon
For IOS

You should upload ios icons to ios/GMart/Images.xcassets/AppIcon.appiconset/ with respective icon name and size

For Android

You should upload android icon images to android/app/src/main/res/mipmap_folder_name(mipmap-hdpi, mipmap-mdpi) with respective icon name and size

Change app name
For IOS

In order to change the app name you need to modify the ios/GMart/info.plist and change CFBundleDisplayName key property value.

For Android

In order to change the app name you need to modify the app.json and change the name and displayName properties and also the app_name property from android/app/src/main/res/values/strings.xml.

Change theme color

You can edit the app theme colors by modifying the src/constants/colors.js file.

Admin panel

Responsive Web Admin - order management system. The system has the ability to manage the catalog of products and work with orders.

As part of the architecture we have included the use of React as a complement for admin panel operations.

  • React Version: 18.2.0
  • React Router Dom Version: ^6.15.0
Features
  • Manage Users
  • Manage Categories
  • Manage Products
  • Manage Banners
  • Manage Orders
  • Dashboard statistics about orders
Configuration

Open .env file at the root of the admin panel.

Admin panel will uses Amazon S3 for files upload.

Configure Amazon S3:

  • Replace REACT_APP_AWS_ACCESS_KEY with your Amazon S3 access key id
  • Replace REACT_APP_AWS_SECRET_KEY with your Amazon S3 secret access key
  • Replace REACT_APP_AWS_REGION with your Amazon S3 region
  • Replace REACT_APP_AWS_BUCKET with your Amazon S3 bucket name

Configure API url:

  • Replace REACT_APP_WEB_API_HOST with the hosting url on which your Node API is running

Configure Google API key:

  • Replace REACT_APP_GOOGLE_API_KEY with the your google account API key
Setup

Run the following commands on your terminal to setup GMart Admin Panel on your system.

cd Gmart-admin
npm install
Run admin panel - npm start

Admin Panel Pages

  4. Products

    5. This page will allow admin to manage all Products.
    Along with Add, Edit, Delete Products, Admin will able to set the Visibility of the Product too. To show Products in Mobile Application, It must have status visible.
    Admin can search products by their name. Also the Filter option available, to filter the products by Categories.

  5. Banners

    6. The Mobile Application having the slider in the Home Page. The Banners of that sliders are listed here.

  6. Orders

    7. This page will allow admin to manage all the orders.

    Also you will see two buttons for some orders, as you see in the screenshot.
    Order Delivered: Once the delivery boy delivers the order, he/she can hit this button, and after confirmation:

    • The order status will be changed to delivered
